# Security Policy

RASTGEN helps maintainers detect security-sensitive change signals, but it does not replace a manual security review.

## Reporting a vulnerability

If you discover a security issue, please open a private report if available on GitHub, or contact the maintainer directly.

Please include:

- affected version;
- steps to reproduce;
- expected behavior;
- actual behavior;
- possible impact.

## Scope

Security reports may include:

- unsafe handling of tokens or secrets;
- command injection risks;
- unsafe file handling;
- incorrect GitHub Action behavior;
- false negatives in security-sensitive checks.

## Maintainer response

The maintainer will review valid reports and prepare a fix when needed.
